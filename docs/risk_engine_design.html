Scoring formula
Final Score = min(100, Σ finding_points × dimension_weight × severity_multiplier) // Each finding contributes base points × its dimension weight × severity multiplier // Score is capped at 100. Multiple findings stack additively. // Any CRITICAL finding sets floor score to 81 regardless of other findings.
Dimension weights
DimensionWeightRationale
Security 40%
Highest weight — security breaches cause immediate, irreversible damage. Data exposure, IAM compromise, and network exposure are existential risks.
Availability 30%
Outages directly cost revenue ($5.6M average). Multi-AZ removal, replica reduction, and health check changes can cause immediate production incidents.
Compliance 20%
Regulatory violations result in fines and audit failures. Often overlaps with security findings — weighted separately to ensure compliance-only gaps are surfaced.
Cost 10%
Lowest weight — cost impact is rarely deployment-blocking alone. Reported for FinOps awareness but does not trigger BLOCK verdicts independently.

Verdict override rules
BLOCK_DEPLOYMENT — automatic triggers
Any single CRITICAL finding → automatic BLOCK regardless of total score. Score ≥ 81 → BLOCK. Custom policy rule match → BLOCK.
if any(f.severity == "CRITICAL" for f in findings): verdict = "BLOCK_DEPLOYMENT" if total_score >= 81: verdict = "BLOCK_DEPLOYMENT"
REVIEW_REQUIRED — automatic triggers
Any HIGH finding present + score ≥ 40. Score between 31–80 with no CRITICAL findings. More than 3 MEDIUM findings.
if any(f.severity == "HIGH") and total_score >= 40: verdict = "REVIEW_REQUIRED" if 31 <= total_score <= 80 and no_critical: verdict = "REVIEW_REQUIRED"
SAFE_TO_DEPLOY — conditions
Score 0–30 AND no CRITICAL or HIGH findings. Only MEDIUM/LOW findings present, or no findings at all.
if total_score <= 30 and no_critical and no_high: verdict = "SAFE_TO_DEPLOY"
